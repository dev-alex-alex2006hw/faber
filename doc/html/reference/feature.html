feature

A feature is a typed variable that can be used to parametrize the build process.

synopsis

class faber.feature.feature(*args, **kwds)
__call__(*args, **kwds)

Instantiate a feature value.

__init__(*args, **kwds)

Create a feature. The constructor takes up to three non-keyword arguments (in arbitrary order):

  • a name (string, defaults to ‘’)
  • a set of valid values (set, defaults to None)
  • attributes (int, defaults to 0)

Alternatively, a composite feature may be constructed passing sub-features as keyword arguments:

feature(name, <name>=feature(...), ...)

class faber.feature.value(type, v, condition, **kwds)
__init__(type, v, condition, **kwds)

Construct a feature value. Arguments:

  • type: the feature
  • item: the value (list if this is a ‘multi’ feature, a single value otherwise)
  • condition: a condition in case this is a conditional feature
  • kwds: subfeatures, if this is a composite feature value
class faber.feature.set(*args)

Examples

from faber.feature import *

include = feature('include', multi)
link = feature('link', values=('shared', 'static'))

i1 = include('search', 'path')
l1 = link('shared')

fs = set(i1, l1)
assert fs.include == ('search', 'path')
